English Grammar · Conditionals

Conditional Sentences —
Types 0, 1, 2 & 3

Conditionals are among the most rule-bound structures in English — the tense combinations are fixed, the conjunctions have strict grammar rules, and small changes in wording (was vs were, would vs would have) carry entirely different meanings. This module covers all four types plus every conjunction-specific trap tested in CDS, NDA and AFCAT.

All Four Types CDS · NDA · AFCAT 15 Practice Questions

A conditional sentence describes what happens (or would happen) under certain conditions. It always has an if-clause (condition) and a main/result clause (consequence). The key insight: the type of conditional is determined by reality — whether the condition is a fact, a real future possibility, a present hypothetical, or an impossible past situation. Mastering this determines whether you can correctly identify errors and fill blanks.

Section 1 — The Four Types in Detail
🔵
Type 0 — Zero Conditional (Timeless Facts)
If + Simple Present → Simple Present | Always true
▾

Type 0 expresses universal truths, scientific facts, and permanent conditions — things that are always true whenever the condition is met. Both clauses use the Simple Present tense. The word "when" can replace "if" without changing meaning.

Type 0 Formula
If + Subject + V1 (Simple Present)
→
Subject + V1 (Simple Present)
✦ "when" can replace "if" — same meaning ✦ No modals (will/would) in either clause
Examples
  • If you heat water to 100°C, it boils. ✓
  • If soldiers lack sleep, their performance deteriorates. ✓
  • When metal is exposed to air, it rusts. ✓
  • If you press the trigger, the weapon fires. ✓
Common Error
  • If you heat water, it will boil. → Simple Present, not "will boil" ✓ (Type 0 fact — "will" changes it to Type 1)
🔷
Type 1 — First Conditional (Real Future Possibility)
If + Simple Present → will + V1 | Possible and likely
▾

Type 1 describes a real, possible future situation — the speaker believes the condition could genuinely happen. The if-clause uses Simple Present; the result clause uses will + base verb.

Type 1 Formula
If + Subject + V1 (Simple Present)
→
Subject + will + V1
✦ Can also use: can, may, might, should, must in the result clause ✦ NEVER use "will" in the if-clause
Examples
  • If he trains hard, he will qualify for the NDA. ✓
  • If it rains, the parade will be postponed. ✓
  • If you study this chapter, you may score full marks. ✓
  • If the ceasefire holds, troops will return to base. ✓
⚑ The Golden Rule — NEVER "will" in the if-clause
  • ✗ "If he will train hard, he will qualify." → ✓ "If he trains hard, he will qualify."
  • ✗ "If it will rain, we will cancel." → ✓ "If it rains, we will cancel."
  • This is the single most tested Type 1 error in all three exams
  • Exception: "will" can appear in an if-clause only when it expresses willingness (a polite request), not futurity: "If you will please wait outside…" — here "will" = willingness
🟣
Type 2 — Second Conditional (Unreal Present / Hypothetical)
If + Past Simple → would + V1 | Imaginary or unlikely now
▾

Type 2 describes an unreal, hypothetical, or very unlikely present/future situation. The speaker imagines a different reality — either because the condition is impossible, or because they consider it unlikely. The if-clause uses the Simple Past tense (even though the meaning is present/future).

Type 2 Formula
If + Subject + V2 (Simple Past)
→
Subject + would / could / might + V1
✦ "were" used for ALL persons in the if-clause (subjunctive mood — formal English) ✦ Never "would" in the if-clause
Examples
  • If I were the commander, I would deploy reserves immediately. ✓
  • If she knew the answer, she would tell you. ✓
  • If it were not for the fog, we could launch the strike. ✓
  • If he had more time, he might prepare better. ✓
⚑ "Were" Not "Was" — The Subjunctive Trap
  • In Type 2 conditionals, formal English uses "were" for all persons — including I, he, she, it
  • ✗ "If I was the officer…" → ✓ "If I were the officer…"
  • ✗ "If she was taller…" → ✓ "If she were taller…"
  • This is the subjunctive mood — used for hypothetical/unreal conditions in formal/exam English
  • "Was" is acceptable only in very informal speech — never in CDS/NDA/AFCAT answers
🔴
Type 3 — Third Conditional (Unreal Past / Regret)
If + Past Perfect → would have + V3 | Impossible — past cannot change
▾

Type 3 refers to an impossible past situation — the condition did not happen, and its result is now imagined. This is the conditional of regret, criticism, and speculation about the past. Both clauses require perfect forms.

Type 3 Formula
If + Subject + had + V3 (Past Perfect)
→
Subject + would / could / might have + V3
✦ Can invert: "Had he trained harder, he would have passed." (no "if") ✦ Never "would have" in the if-clause
Examples
  • If he had trained harder, he would have passed the selection. ✓
  • If we had received the intelligence in time, we could have prevented the attack. ✓
  • If the officer had followed orders, casualties would have been minimal. ✓
  • Had he arrived earlier, he would have met the general. ✓ (inverted)
The Two Most Common Type 3 Errors
  • If he had trained harder, he would pass. → "would have passed" (result needs "have + V3") ✗
  • If he would have trained, he would have passed. → "would have" in if-clause is always wrong ✗
Section 2 — Mixed Conditionals
🔀
Mixed Conditionals — Crossing Past & Present
Past condition + present result, or present condition + past result
▾

Sometimes a conditional sentence mixes time frames — the condition and the result belong to different times. Two patterns are tested.

TypeIf-clauseResult ClauseMeaning
Past → PresentPast Perfect (had + V3)would + V1A past condition affects the present
Present → PastSimple Past / werewould have + V3A present state would have affected a past event
Mixed Conditional Examples
  • If he had joined the army [past], he would be an officer now. [present] ✓ (Past → Present)
  • If I were braver [present], I would have volunteered for that mission. [past] ✓ (Present → Past)
How to Identify Mixed Conditionals

Look for time markers: "now", "today", "currently" in the result clause signal Present → the condition may be past. Conversely, a past event result ("would have…") with a present-state if-clause signals the Present → Past mix.

Section 3 — Conditional Conjunctions & Their Rules
🔗
Other Conditional Conjunctions — Unless, Until, As long as & More
Each has its own grammar rule — these are heavily tested in Spotting Errors
▾

Beyond "if", several other conjunctions introduce conditional clauses. Each has a fixed grammar rule — and its own common error pattern.

UNLESS = "if…not"
Never use "not" after "unless" — it already contains negation. Follow same tense rules as "if".
✓ "Unless you train hard, you will not qualify."  |  ✗ "Unless you do not train…" (double negative)
PROVIDED / PROVIDED THAT / ON CONDITION THAT = "if and only if"
Means a strict condition. Uses Simple Present in the conditional clause — never "will".
✓ "You may enter provided you show your ID."  |  ✗ "provided you will show…"
AS LONG AS / SO LONG AS = "on the condition that"
Uses Simple Present in the clause — never future tense inside the clause.
✓ "As long as you follow orders, you will succeed."  |  ✗ "as long as you will follow…"
IN CASE = "because there is a possibility"
Expresses precaution. Always uses Simple Present (not "will") in the in-case clause.
✓ "Take a raincoat in case it rains."  |  ✗ "in case it will rain"
SUPPOSE / SUPPOSING = "what if" (hypothetical)
Introduces Type 2 / hypothetical conditions. Followed by Past Simple or Past Perfect.
✓ "Suppose he were wrong, what would we do?"  |  ✓ "Supposing she had known, would she have told us?"
BUT FOR = "if it were not for / if it had not been for"
Used with Type 2 (but for + noun → would + V1) or Type 3 (but for + noun → would have + V3). No verb follows "but for" directly.
✓ "But for his bravery, we would have lost." (Type 3)  |  ✓ "But for the fog, we could see clearly." (Type 2)
EVEN IF = "regardless of whether"
Same tense rules as "if" — follow Type 1, 2, or 3 structure. Adds emphasis that the result holds whatever the condition.
✓ "Even if it rains, we will march." (Type 1)  |  ✓ "Even if he had tried, he would have failed." (Type 3)
ONLY IF = strict condition (emphatic)
When "only if" starts a sentence, the main clause is inverted: "Only if + condition, auxiliary + subject + verb".
✓ "Only if you train daily will you improve." (inverted main clause)  |  ✗ "Only if you train daily you will improve."
⚑ The No-Future-in-Conditional-Clause Rule
  • All conditional/time-clause conjunctions — if, unless, provided, as long as, in case, when, before, after, until — share one rule:
  • Never use "will/shall/would" inside the conditional/time clause itself.
  • The future tense goes only in the main (result) clause.
  • ✗ "Unless you will study…" → ✓ "Unless you study…"
  • ✗ "Provided you will attend…" → ✓ "Provided you attend…"
Section 4 — Inverted Conditionals (Without "If")
🔄
Formal Inversion — Dropping "If" & Inverting the Verb
Were I… / Had he… / Should you… — formal/literary style
▾

In formal and literary English, "if" can be omitted from conditional clauses if the auxiliary verb is placed before the subject. This inversion is tested in CDS/NDA fill-in-the-blanks and spotting errors.

Normal FormInverted Formal FormType
If he were ready…Were he ready…Type 2
If he had known…Had he known…Type 3
If you should need help…Should you need help…Type 1 (tentative)
If it were not for…Were it not for…Type 2
If it had not been for…Had it not been for…Type 3
Inverted Conditional Examples
  • Were I the general, I would order a full retreat. ✓ (= If I were…)
  • Had the officer received the signal, reinforcements would have arrived. ✓ (= If the officer had…)
  • Should you require assistance, contact the control room. ✓ (= If you should…)
Conditionals — Complete Quick Reference
The Four Types at a Glance
  • Type 0: If + Pres. Simple → Pres. Simple  (universal fact)
  • Type 1: If + Pres. Simple → will + V1  (real future)
  • Type 2: If + Past Simple (were) → would + V1  (hypothetical present)
  • Type 3: If + Past Perfect → would have + V3  (unreal past)
  • Mixed (past→present): If + Past Perfect → would + V1
  • Mixed (present→past): If + Past Simple/were → would have + V3
Golden Rules — Never Violate These
  • ✗ Never "will" in the if-clause (Types 1, 0)
  • ✗ Never "would" in the if-clause (Types 2, 3)
  • ✗ Never "would have" in the if-clause (Type 3)
  • ✓ Type 2 if-clause: always "were" — never "was" in formal English
  • ✓ Type 3 result: always "would/could/might have + V3"
  • ✓ "Unless" already means "if not" — never add "not" after "unless"
Conjunction Grammar Rules
  • Unless: = if…not → never "unless…not"
  • Provided / as long as / in case: → Simple Present inside (no will)
  • But for + noun: Type 2 → would; Type 3 → would have
  • Suppose / supposing: followed by Past / Past Perfect
  • Even if: same tense rules as "if"
  • Only if (sentence-initial): main clause must be inverted
Inverted Conditionals (No "If")
  • Were + subject → Type 2: "Were I the commander…"
  • Had + subject + V3 → Type 3: "Had he known…"
  • Should + subject → Type 1 (tentative): "Should you need…"
  • Were it not for → Type 2 version of "but for"
  • Had it not been for → Type 3 version of "but for"
  • Inversion = formal/literary; tested in fill-in-the-blanks
Spotting Error Checklist
  • 1. Identify the conditional type (0/1/2/3/mixed)
  • 2. Check if-clause tense — is "will/would" wrongly placed there?
  • 3. Check result clause — does it match the type's formula?
  • 4. If Type 2 — is "were" used (not "was")?
  • 5. If Type 3 — does result have "have + V3" (not just "would + V1")?
  • 6. If "unless" — is there an extra "not" after it?
  • 7. If "provided / as long as / in case" — is there a "will" inside?
Result Clause Variations
  • Type 1 result: will / can / may / might / should / must + V1
  • Type 2 result: would / could / might + V1
  • Type 3 result: would / could / might + have + V3
  • Imperative as result (Type 1): "If you finish early, leave."
  • Present tense as result (Type 0/habitual): "If he's late, we start without him."
